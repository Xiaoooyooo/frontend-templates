## 开发注意事项

### 文件目录结构

以下为项目初始结构，你可以根据项目实际需求进行调整和补充。

├─index.html 网站入口 HTML 文件
├─public 公共资源目录，如 favicon.ico 等
├─src 项目源代码目录
│ ├─assets 静态资源目录，如样式、图片、字体等
│ │ └─style
│ ├─components 组件目录，包含所有的可复用组件
│ │ └─layout 布局组件目录，包含页面布局相关的组件
│ ├─context 自定义React Context目录
│ ├─lib 库目录，包含所有的工具函数、自定义钩子、API等
│ │ ├─api API客户端目录，包含所有的后端请求封装
│ │ ├─hooks 自定义钩子目录，包含所有的自定义钩子函数
│ │ └─utils 工具函数目录，包含所有的通用工具函数
│ ├─router 路由目录，包含所有的路由配置
│ ├─store 状态管理目录，包含所有的状态管理相关代码
│ └─views 视图目录，包含所有的页面组件

### HTTP请求

1. 所有对后端的请求都通过统一的请求客户端（`src/lib/api/client.ts`）进行，以确保统一的请求格式和错误处理，不得使用其他第三方库（如axios）
2. 所有功能模块的请求都需要在使用之前进行封装，并存放在`src/lib/api`目录下，每个功能模块对应一个文件
3. 务必要使用`useFetch`（`src/lib/hooks/useFetch.ts`）钩子结合已封装的请求函数进行远程数据获取

### 编码规范和风格

1. 该项目使用 eslint + prettier 进行代码风格检查和格式化，务必要严格遵守根路径下的`.prettierrc`配置进行编码
2. 使用意义相近的英文单词命名变量和常量，变量使用驼峰命名法，常量使用下划线分隔的全大写单词
3. 相同的代码不得出现两次，转而将其封装成公共函数或组件，以提高代码的可维护性和复用性
4. 严格遵守组件化开发规范，每一个独立的功能模块都应该是一个独立的组件，包含其自己的状态、逻辑和渲染，不可在一个组件中写完整个页面逻辑

### 常用命令

- `pnpm run lint` 格式化代码
- `pnpm run dev` 启动开发服务器
- `pnpm run build` 构建生产版本
- `pnpm run preview` 预览生产版本
